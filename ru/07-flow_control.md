# 7. Управление потоком и условные циклы

---

## 7.1 Объединение команд: суть Shell

Одиночные команды имеют ограниченные возможности. Только **объединяя** их, можно выполнять сложные задачи.

ИИ мощён во многом потому, что владеет этими комбинациями:

```bash
cat access.log | grep "ERROR" | sort | uniq -c | sort -rn | head -10
```

Это означает: «Из access.log найти ошибки, подсчитать вхождения, показать топ-10»

---

## 7.2 `|` (pipe): Искусство потока данных

Труба превращает **вывод** предыдущей команды во **ввод** следующей.

```bash
# Сортировать содержимое файла
cat unsorted.txt | sort

# Найти самые частые команды
history | awk '{print $2}' | sort | uniq -c | sort -rn | head -10

# Извлечь IP из лога и подсчитать
cat access.log | awk '{print $1}' | sort | uniq -c | sort -rn | head -5
```

### Перенаправление stderr в трубу

```bash
# Отправить stderr в трубу
command1 2>&1 | command2

# Или в Bash 4+
command1 |& command2
```

## 7.3 `&&`: Выполнить следующее только при успехе

**Только если `command1` успешно (код завершения = 0), `command2` выполнится.**

```bash
# Создать каталог, затем перейти в него
mkdir -p project && cd project

# Компилировать, затем запускать
gcc -o program source.c && ./program

# Скачать, затем распаковать
curl -L -o archive.tar.gz http://example.com/file && tar -xzf archive.tar.gz
```

---

## 7.4 `||`: Выполнить следующее только при неудаче

**Только если `command1` неудачно (код завершения ≠ 0), `command2` выполнится.**

```bash
# Создать файл, если его нет
[ -f config.txt ] || echo "Конфиг отсутствует" > config.txt

# Попробовать один способ, fallback к другому
cd /opt/project || cd /home/user/project

# Обеспечить успех даже при неудаче (часто в makefile)
cp file.txt file.txt.bak || true
```

### Комбинирование `&&` и `||`

```bash
# Условное выражение
[ -f config ] && echo "Найден" || echo "Не найден"

# Эквивалентно:
if [ -f config ]; then
    echo "Найден"
else
    echo "Не найден"
fi
```

## 7.5 `;`: Выполнить независимо

```bash
# Все три выполнятся
mkdir /tmp/test ; cd /tmp/test ; pwd
```

## 7.6 `$()`: Подстановка команды

**Выполнить команду, заменить `$()` её выводом.**

```bash
# Базовое использование
echo "Сегодня $(date +%Y-%m-%d)"
# Вывод: Сегодня 2026-03-22

# В переменных
FILES=$(ls *.txt)

# Получить имя каталога
DIR=$(dirname /path/to/file.txt)
BASE=$(basename /path/to/file.txt)

# Вычислить
echo "Результат $((10 + 5))"
# Вывод: Результат 15
```

### vs Обратные кавычки

```bash
# Оба эквивалентны
echo "Сегодня $(date +%Y)"
echo "Сегодня `date +%Y`"

# Но $() лучше, потому что может вкладываться
echo $(echo $(echo вложено))
```

## 7.7 `[[ ]]` и `[ ]`: Условные тесты

### Тесты файлов

```bash
[[ -f file.txt ]]      # Обычный файл существует
[[ -d directory ]]     # Каталог существует
[[ -e path ]]           # Любой тип существует
[[ -L link ]]           # Символическая ссылка существует
[[ -r file ]]           # Читаемый
[[ -w file ]]           # Записываемый
[[ -x file ]]           # Исполняемый
[[ file1 -nt file2 ]]  # file1 новее file2
```

### Тесты строк

```bash
[[ -z "$str" ]]        # Строка пустая
[[ -n "$str" ]]        # Строка не пустая
[[ "$str" == "value" ]] # Равны
[[ "$str" =~ pattern ]]  # Совпадение с regex
```

### Числовые тесты

```bash
[[ $num -eq 10 ]]      # Равно
[[ $num -ne 10 ]]      # Не равно
[[ $num -gt 10 ]]      # Больше
[[ $num -lt 10 ]]      # Меньше
```

## 7.8 `if`: Условные операторы

```bash
if [[ условие ]]; then
    # сделать что-то
elif [[ условие2 ]]; then
    # сделать что-то другое
else
    # резервный вариант
fi
```

### Полный пример

```bash
#!/bin/bash

FILE="config.yaml"

if [[ ! -f "$FILE" ]]; then
    echo "Ошибка: $FILE не существует"
    exit 1
fi

if [[ -r "$FILE" ]]; then
    echo "Файл читаемый"
else
    echo "Файл не читаемый"
fi
```

## 7.9 `for`: Циклы

### Базовая syntax

```bash
for variable in list; do
    # использовать $variable
done
```

### Частые паттерны ИИ

```bash
# Обработать все .txt файлы
for file in *.txt; do
    echo "Обработка $file"
done

# Диапазон чисел
for i in {1..10}; do
    echo "Итерация $i"
done

# Массив
for color in красный зелёный синий; do
    echo $color
done

# Цикл в стиле C (Bash 3+)
for ((i=0; i<10; i++)); do
    echo $i
done
```

---

## 7.10 `while`: Условные циклы

```bash
# Читать строки
while IFS= read -r line; do
    echo "Прочитано: $line"
done < file.txt

# Счётный цикл
count=0
while [[ $count -lt 10 ]]; do
    echo $count
    ((count++))
done
```

## 7.11 `case`: Сопоставление с образцом

```bash
case $ACTION in
    start)
        echo "Запуск сервиса..."
        ;;
    stop)
        echo "Остановка сервиса..."
        ;;
    restart)
        $0 stop
        $0 start
        ;;
    *)
        echo "Использование: $0 {start|stop|restart}"
        exit 1
        ;;
esac
```

### Подстановочные знаки

```bash
case "$filename" in
    *.txt)
        echo "Текстовый файл"
        ;;
    *.jpg|*.png|*.gif)
        echo "Изображение"
        ;;
    *)
        echo "Неизвестный тип"
        ;;
esac
```

## 7.12 Краткая справка

| Символ | Название | Описание |
|--------|----------|----------|
| `\|` | Труба | Передать вывод следующему вводу |
| `&&` | И | Выполнить следующее только если предыдущее успешно |
| `\|\|` | ИЛИ | Выполнить следующее только если предыдущее неудачно |
| `;` | Точка с запятой | Выполнить независимо |
| `$()` | Подстановка команды | Выполнить, заменить выводом |
| `[[ ]]` | Условный тест | рекомендуемый синтаксис тестов |
| `if` | Условие | ветвление на основе условия |
| `for` | Счётный цикл | итерация по списку |
| `while` | Условный цикл | повторять пока условие истинно |
| `case` | Сопоставление образцу | многоходовое ветвление |

## 7.13 Упражнения

1. Используйте `|` для объединения `ls`, `grep`, `wc` для подсчёта `.log` файлов
2. Используйте `&&` чтобы убедиться, что `cd` успешен перед продолжением
3. Используйте цикл `for` для создания 10 каталогов (dir1 до dir10)
4. Используйте `while read` для чтения и отображения /etc/hosts
5. Напишите простой калькулятор с `case` (сложение, вычитание, умножение, деление)
