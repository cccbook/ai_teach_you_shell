# 5. Обработка текста

---

## 5.1 Философия обработки текста ИИ

В мире ИИ **всё — это текст**.

- Код — это текст
- Конфигурационные файлы — это текст
- Логи — это текст
- JSON, HTML, Markdown — всё это текст

Поэтому команды обработки текста — основа инструментария ИИ.

Когда инженеры-люди сталкиваются с проблемами: «Мне нужен инструмент для этого...»
Когда ИИ сталкивается с проблемами: «Это решается `grep | sed | awk` в одну строку.»

---

## 5.2 `cat`: Искусство чтения файлов

### Базовое использование

```bash
# Показать содержимое файла
cat file.txt

# Объединить файлы
cat part1.txt part2.txt > whole.txt

# Показать с номерами строк
cat -n script.sh
```

### Настоящее назначение: объединение и создание

```bash
cat << 'EOF' > newfile.txt
Содержимое файла
Можно записать много строк
EOF
```

## 5.3 `head` и `tail`: Смотреть только нужное

### `head`: Посмотреть начало

```bash
# Первые 10 строк (по умолчанию)
head file.txt

# Первые 5 строк
head -n 5 file.txt

# Первые 100 байт
head -c 100 file.txt
```

### `tail`: Посмотреть конец

```bash
# Последние 10 строк (по умолчанию)
tail file.txt

# Последние 5 строк
tail -n 5 file.txt

# Следить за файлом в реальном времени (самое частое!)
tail -f /var/log/syslog

# Следить и фильтровать
tail -f app.log | grep --line-buffered ERROR
```

### Просмотр определённого диапазона строк

```bash
# Просмотреть строки 100-150
tail -n +100 file.txt | head -n 50
```

## 5.4 `wc`: Инструмент подсчёта

```bash
# Подсчитать строки
wc -l file.txt

# Подсчитать строки для нескольких файлов
wc -l *.py

ls | wc -l
```

## 5.5 `grep`: Король текстового поиска

### Базовое использование

```bash
# Найти строки с "error"
grep "error" log.txt

# Игнорировать регистр
grep -i "error" log.txt

# Показать номера строк
grep -n "error" log.txt

# Показать только имена файлов
grep -l "TODO" *.md

# Инвертировать (несовпадающие строки)
grep -v "debug" log.txt

# Совпадение целого слова
grep -w "error" log.txt
```

### Регулярные выражения

```bash
# Совпадение в начале
grep "^Error" log.txt

# Совпадение в конце
grep "done.$" log.txt

# Любой символ
grep "e.or" log.txt

# Диапазон
grep -E "[0-9]{3}-" log.txt
```

### Продвинутые техники

```bash
# Рекурсивный поиск
grep -r "TODO" src/

# Только определённое расширение
grep -r "TODO" --include="*.py" src/

# Показать контекстные строки
grep -B 2 -A 2 "ERROR" log.txt

# Несколько условий (ИЛИ)
grep -E "error|warning|fatal" log.txt
```

## 5.6 `sed`: Инструмент замены текста

### Базовая замена

```bash
# Заменить первое совпадение
sed 's/старое/новое/' file.txt

# Заменить все совпадения
sed 's/старое/новое/g' file.txt

# Замена на месте
sed -i 's/старое/новое/g' file.txt

# Сначала резервная копия, потом замена
sed -i.bak 's/старое/новое/g' file.txt
```

### Удаление строк

```bash
# Удалить пустые строки
sed '/^$/d' file.txt

# Удалить строки комментариев
sed '/^#/d' file.txt

# Удалить конечные пробелы
sed 's/[[:space:]]*$//' file.txt
```

### Практические примеры

```bash
# Пакетное переименование расширений (.txt → .md)
for f in *.txt; do
    mv "$f" "$(sed 's/.txt$/.md/' <<< "$f")"
done

# Удалить окончания строк Windows
sed -i 's/\r$//' file.txt
```

## 5.7 `awk`: Швейцарский нож обработки текста

### Базовая концепция

`awk` обрабатывает текст строка за строкой, автоматически разделяя на поля ($1, $2, $3...), выполняя указанные действия для каждой строки.

### Базовое использование

```bash
# Разделение по умолчанию (пробелы)
awk '{print $1}' file.txt

# Указать разделитель
awk -F: '{print $1}' /etc/passwd

# Вывести несколько полей
awk -F: '{print $1, $3, $7}' /etc/passwd
```

### Условная обработка

```bash
# Обрабатывать только совпадающие строки
awk -F: '$3 > 1000 {print $1}' /etc/passwd

# BEGIN и END
awk 'BEGIN {print "Начало"} {print} END {print "Конец"}' file.txt
```

### Практические примеры

```bash
# Сумма столбца CSV
awk -F, '{sum += $3} END {print sum}' data.csv

# Найти максимум
awk 'NR==1 {max=$3} $3>max {max=$3} END {print max}' data.csv

# Форматированный вывод
awk '{printf "%-20s %10.2f\n", $1, $2}' data.txt
```

## 5.8 Практика: Объединение всех инструментов

### Сценарий: Анализ логов сервера

```bash
# 1. Найти сообщения об ошибках
grep -i "error" access.log

# 2. Подсчитать ошибки
grep -ci "error" access.log

# 3. Найти самые частые ошибки
grep "error" access.log | awk '{print $NF}' | sort | uniq -c | sort -rn | head

# 4. Подсчитать запросы по часам
awk '{print $4}' access.log | cut -d: -f2 | sort | uniq -c
```

### Сценарий: Пакетное изменение кода

```bash
# Заменить "print" на "logger.info" во всех .py файлах
find . -name "*.py" -exec sed -i 's/print(/logger.info(/g' {} +

# Заменить var на const
find . -name "*.js" -exec sed -i 's/\bvar\b/const/g' {} +
```

## 5.9 Краткая справка

| Команда | Назначение | Частые флаги |
|---------|------------|--------------|
| `cat` | Показать/объединить файлы | `-n` номера строк |
| `head` | Просмотр начала файла | `-n` строки, `-c` байты |
| `tail` | Просмотр конца файла | `-n` строки, `-f` следить |
| `wc` | Подсчёт | `-l` строки, `-w` слова, `-c` байты |
| `grep` | Текстовый поиск | `-i` игнорировать, `-n` номер строки, `-r` рекурсивно, `-c` подсчёт |
| `sed` | Замена текста | `s/старое/новое/g`, `-i` на месте |
| `awk` | Обработка полей | `-F` разделитель, `{print}` действие |

---

## 5.10 Упражнения

1. Используйте `head` и `tail` для просмотра строк 100-120
2. Используйте `grep` для поиска всех пользователей с `/bin/bash` в /etc/passwd
3. Используйте `sed` для замены всех `\r\n` на `\n`
4. Используйте `awk` для вычисления максимума, минимума и среднего значения в числовом файле
