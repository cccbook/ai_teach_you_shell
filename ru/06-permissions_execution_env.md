## 6.1 `chmod`: Искусство прав доступа

### Основы прав в Linux/Unix

```
drwxr-xr-x  3 ai  staff   96 Mar 22 10:30 .
-rw-r--r--  1 ai  staff  4096 Mar 22 10:30 README.md
-rwxr-xr-x  1 ai  staff   128 Mar 22 10:30 script.sh
```

9 символов делятся на три группы:
- `rwx` (владелец): чтение, запись, выполнение
- `r-x` (группа): чтение, выполнение
- `r--` (остальные): только чтение

### chmod: два представления

**Числовое (восьмеричное)**:

```
r = 4, w = 2, x = 1

rwx = 7, r-x = 5, r-- = 4
```

Частые комбинации:
- `777` = rwxrwxrwx (опасно!)
- `755` = rwxr-xr-x
- `644` = rw-r--r--
- `700` = rwx------
- `600` = rw-------

**Символьное**:

```bash
chmod u+x script.sh    # Владелец добавляет выполнение
chmod g-w file.txt     # Группа убирает запись
chmod +x script.sh     # Все добавляют выполнение
```

### Частое использование chmod ИИ

```bash
# Сделать скрипт исполняемым (почти каждый скрипт)
chmod +x script.sh

# Сделать каталог проходимым
chmod +x ~/projects

# Каталоги с записью для группы
chmod -R g+w project/
```

## 6.2 Выполнение Shell-скриптов

### Способы выполнения

```bash
# Способ 1: Выполнение по пути (нужно право на выполнение)
./script.sh

# Способ 2: Через bash (право на выполнение не нужно)
bash script.sh

# Способ 3: Через source (выполняется в текущей оболочке)
source script.sh
```

### Когда что использовать?

| Способ | Когда использовать | Особенности |
|--------|-------------------|-------------|
| `./script.sh` | Стандартное выполнение | нужно `chmod +x`, дочерняя оболочка |
| `bash script.sh` | Указать оболочку | право на выполнение не нужно |
| `source script.sh` | Установить переменные среды | выполняется в текущей оболочке |

### Ключевое различие: `source` vs `./script`

```bash
# Содержимое script.sh: export MY_VAR="привет"

# Выполнить с ./
./script.sh
echo $MY_VAR  # Вывод: (пусто) ← в дочерней оболочке, переменная потеряна

# Выполнить с source
source script.sh
echo $MY_VAR  # Вывод: привет ← в текущей оболочке, переменная осталась
```

## 6.3 `export`: Переменные окружения

```bash
# Установить переменную окружения
export NAME="Алиса"
export PATH="$PATH:/новый/каталог"

# Показать все переменные окружения
export

# Частые переменные
echo $HOME      # Домашний каталог
echo $USER      # Имя пользователя
echo $PATH      # Путь поиска
echo $PWD       # Текущий каталог
```

### Сохранение переменных окружения

```bash
# Добавить в ~/.bashrc
echo 'export EDITOR=vim' >> ~/.bashrc

# Применить изменения
source ~/.bashrc
```

## 6.4 `source`: Загрузка файлов

Эквивалент: напрямую **вставить** содержимое файла в текущую позицию и выполнить.

### Частое использование

```bash
# Загрузить виртуальное окружение
source venv/bin/activate

# Загрузить .env файл
source .env

# Загрузить библиотеку
source ~/scripts/common.sh
```

### Практика: Модульные конфигурационные файлы

```bash
# config.sh
export DB_HOST="localhost"
export DB_PORT="5432"

# Использовать в других скриптах
source config.sh
psql -h $DB_HOST -p $DB_PORT
```

## 6.5 `env`: Управление средой

```bash
# Показать все переменные окружения
env

# Выполнить в чистой среде
env -i HOME=/tmp PATH=/bin sh

# Установить переменные и выполнить
env VAR1=value1 VAR2=value2 ./my_program
```

### Поиск команд

```bash
which python      # Найти расположение команды
type cd           # Найти встроенные команды оболочки
whereis gcc       # Найти все связанные файлы
```

## 6.6 `sudo`: Повышение привилегий

```bash
# Выполнить от root
sudo rm /var/log/old.log

# Выполнить от конкретного пользователя
sudo -u postgres psql

# Показать, что можно делать
sudo -l
```

### Предупреждение об опасности

```bash
# Никогда не выполняйте это!
sudo rm -rf /

# Никогда не делайте это!
sudo curl http://unknown-site.com | sh
```

## 6.7 Краткая справка

| Команда | Назначение | Частые флаги |
|---------|------------|--------------|
| `chmod` | Изменить права файла | `+x` добавить выполнение, `755` восьмеричное, `-R` рекурсивно |
| `chown` | Изменить владельца | `user:group`, `-R` рекурсивно |
| `./script` | Выполнить скрипт (нужно x) | - |
| `bash script` | Выполнить скрипт (x не нужно) | - |
| `source` | Выполнить в текущей оболочке | - |
| `export` | Установить переменные окружения | `-n` удалить |
| `env` | Показать/управлять средой | `-i` очистить |
| `sudo` | Выполнить от root | `-u user` указать пользователя |

## 6.8 Упражнения

1. Создайте скрипт, установите права с `chmod 755`, затем выполните его
2. Выполните один и тот же скрипт с переменными окружения с помощью `source` и `./`, наблюдайте разницу
3. Используйте `env -i` для создания чистой среды, выполните `python --version`
4. Создайте файл `.env`, используйте `source` для загрузки его переменных
