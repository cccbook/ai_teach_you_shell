# 9. การจัดการข้อผิดพลาด

---

## 9.1 ทำไมการจัดการข้อผิดพลาดจึงสำคัญ

ไม่มีการจัดการข้อผิดพลาด:
- เมื่อคำสั่งหนึ่งล้มเหลว จะทำงานต่อด้วยการดำเนินการที่ผิดพลาด
- อาจลบไฟล์ผิด
- อาจเขียนทับข้อมูลสำคัญ
- อาจทิ้งระบบไว้ในสถานะไม่สอดคล้องกัน

มีการจัดการข้อผิดพลาด:
- หยุดทันทีเมื่อเกิดข้อผิดพลาด
- แสดงข้อความข้อผิดพลาดที่มีความหมาย
- ทำความสะอาดก่อนออก

---

## 9.2 รหัสออก (Exit Codes)

ทุกคำสั่งจะส่งคืนรหัสออกหลังการทำงาน:

- `0`: สำเร็จ
- `non-zero`: ล้มเหลว

```bash
# ตรวจสอบรหัสออกของคำสั่งล่าสุด
ls /tmp
echo $?  # Output: 0 (ถ้าสำเร็จ)

ls /nonexistent
echo $?  # Output: 2 (ถ้าล้มเหลว)
```

## 9.3 `set -e`: หยุดเมื่อเกิดข้อผิดพลาด

```bash
#!/bin/bash
set -e

mkdir -p backup
cp important.txt backup/  # ถ้าล้มเหลว สคริปต์จะหยุดที่นี่
rm important.txt          # จะไม่ถึงที่นี่
```

### เมื่อไหร่ควรใช้

เกือบทุกสคริปต์ควรใช้ `set -e`:
- สคริปต์เริ่มต้นระบบ
- สคริปต์ deploy
- สคริปต์ทดสอบอัตโนมัติ

---

## 9.4 `set -u`: ข้อผิดพลาดเมื่อตัวแปรไม่ได้กำหนด

```bash
#!/bin/bash
set -u

echo $undefined_var
# Output: bash: undefined_var: unbound variable
```

### การใช้ร่วมกัน

```bash
#!/bin/bash
set -euo pipefail

# -e: หยุดเมื่อเกิดข้อผิดพลาด
# -u: ข้อผิดพลาดเมื่อตัวแปรไม่ได้กำหนด
# -o pipefail: pipe ล้มเหลวถ้าคำสั่งใดล้มเหลว
```

**นี่คือส่วนหัวมาตรฐานของสคริปต์ AI!**

## 9.5 `trap`: การจัดการข้อผิดพลาดอย่างสวยงาม

### การใช้งานพื้นฐาน

```bash
#!/bin/bash
set -euo pipefail

cleanup() {
    echo "Cleaning up..."
    rm -f /tmp/tempfile
}

trap cleanup EXIT

# โปรแกรมหลัก
echo "Starting process..."
```

### จับข้อผิดพลาด

```bash
#!/bin/bash
set -euo pipefail

error_handler() {
    local exit_code=$?
    echo "❌ สคริปต์ล้มเหลวที่บรรทัด $1, รหัสออก: $exit_code"
    exit $exit_code
}

trap 'error_handler $LINENO' ERR
```

## 9.6 ฟังก์ชันข้อผิดพลาดแบบกำหนดเอง

```bash
#!/bin/bash
set -euo pipefail

die() {
    echo "❌ $@" >&2
    exit 1
}

warn() {
    echo "⚠️ $@"
}

need_command() {
    command -v "$1" &>/dev/null || die "ต้องการคำสั่ง: $1"
}

need_file() {
    [[ -f "$1" ]] || die "ต้องการไฟล์: $1"
}
```

## 9.7 สรุปย่อ

| คำสั่ง | คำอธิบาย |
|--------|-------------|
| `$?` | รหัสออกของคำสั่งล่าสุด |
| `set -e` | หยุดเมื่อเกิดข้อผิดพลาด |
| `set -u` | ข้อผิดพลาดเมื่อตัวแปรไม่ได้กำหนด |
| `set -o pipefail` | pipe ล้มเหลวถ้าคำสั่งใดล้มเหลว |
| `set -euo pipefail` | รวมกัน (แนะนำ) |
| `trap 'func' EXIT` | ทำงานเมื่อออก |
| `trap 'func' ERR` | ทำงานเมื่อเกิดข้อผิดพลาด |
| `trap 'func' INT` | ทำงานเมื่อกด Ctrl+C |
| `exit 1` | ออกพร้อมรหัสข้อผิดพลาด 1 |

---

## 9.8 แบบฝึกหัด

1. เขียนสคริปต์ที่มี `set -euo pipefail` แสดง "Error occurred" เมื่อเกิดความล้มเหลว
2. สร้างฟังก์ชัน `die()` ที่พิมพ์ข้อความแล้วออก
3. ใช้ `trap` เพื่อแสดง "Goodbye" เมื่อสคริปต์ออก
4. เขียนสคริปต์ deploy ที่มีการย้อนกลับอัตโนมัติเมื่อล้มเหลว
