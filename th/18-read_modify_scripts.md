# 18. การอ่านและแก้ไข Script ของคนอื่น

---

## 18.1 ทำไมต้องอ่าน Script ของคนอื่น

- รับช่วงดูแลโปรเจกต์
- ใช้เครื่องมือ open source
- แก้ไขปัญหา
- เรียนรู้เทคนิคใหม่

AI พบเจอสคริปต์ที่ไม่คุ้นเคยทุกวัน ดังนั้นการเรียนรู้ที่จะเข้าใจ Shell scripts ของคนอื่นอย่างรวดเร็วจึงเป็นสิ่งจำเป็น

---

## 18.2 การสังเกตเบื้องต้น

### ขั้นตอน 1: ตรวจ shebang

```bash
head -1 script.sh
```

```bash
#!/bin/bash      # ใช้ bash
#!/bin/sh        # ใช้ POSIX sh (เข้ากันได้มากกว่า)
```

### ขั้นตอน 2: ตรวจสิทธิ์และขนาด

```bash
ls -la script.sh
wc -l script.sh
```

### ขั้นตอน 3: ตรวจสอบไวยากรณ์อย่างรวดเร็ว

```bash
bash -n script.sh  # ตรวจสอบไวยากรณ์เท่านั้น ไม่รัน
```

---

## 18.3 เข้าใจโครงสร้าง

### โครงสร้างทั่วไป

```bash
#!/bin/bash
# Comment: คำอธิบายสคริปต์

# ตั้งค่า
set -euo pipefail
VAR="value"

# กำหนดฟังก์ชัน
function help() { ... }

# การทำงานหลัก
main() { ... }

# รัน
main "$@"
```

### หาการทำงานหลัก

```bash
# ดูบรรทัดสุดท้าย
tail -20 script.sh

# หาการกำหนดฟังก์ชัน
grep -n "^function\|^[[:space:]]*[a-z_]*\(" script.sh
```

---

## 18.4 คำสั่งวิเคราะห์ที่ใช้บ่อย

```bash
# หาการกำหนดฟังก์ชันทั้งหมด
grep -n "^[[:space:]]*function" script.sh

# หาเงื่อนไขทั้งหมด
grep -n "if\|\[\[" script.sh

# หาลูปทั้งหมด
grep -n "for\|while\|do\|done" script.sh

# หา command substitutions
grep -n '\$(' script.sh

# หาทุกการออก
grep -n "exit" script.sh
```

---

## 18.5 การตรวจสอบความปลอดภัย

```bash
# หาคำสั่งอันตราย
grep -n "rm -rf" script.sh
grep -n "sudo" script.sh
grep -n "eval" script.sh

# ตรวจความเสี่ยงจากการฉีดตัวแปร
grep -n '\$[A-Za-z_][A-Za-z0-9_]*[^"}]' script.sh
```

---

## 18.6 แบบฝึกหัด

1. วิเคราะห์ Shell script ที่มีอยู่ด้วย `grep` และ `awk`
2. หาตัวแปรทั้งหมดในสคริปต์และเข้าใจวัตถุประสงค์
3. ใช้ `bash -n` ตรวจสอบไวยากรณ์ของสคริปต์
4. เพิ่มคำอธิบายให้สคริปต์ที่ไม่มีคอมเมนต์ โดยอธิบายแต่ละส่วน
