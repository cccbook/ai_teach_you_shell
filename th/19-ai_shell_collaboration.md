# 19. โหมดการทำงานร่วมกันของ AI + Shell

---

## 19.1 รูปแบบใหม่ของการทำงานร่วมกันระหว่างมนุษย์และเครื่อง

การเขียนโปรแกรมแบบดั้งเดิม:
- มนุษย์พิมพ์ด้วยแป้นพิมพ์
- มนุษย์ใช้เมาส์ในการทำงานกับ IDE
- มนุษย์รันและทดสอบ

การเขียนโปรแกรมในยุค AI:
- มนุษย์อธิบายความต้องการ
- AI สร้างคำสั่ง Shell และสคริปต์
- มนุษย์ตรวจสอบและรัน
- มนุษย์และ AI debug ด้วยกัน

---

## 19.2 การอธิบายความต้องการให้ AI

### คำอธิบายที่ดี

```bash
# งานที่ชัดเจน
"Find all .log files larger than 100MB in /var/log"

# ระบุรูปแบบ output ที่คาดหวัง
"List all .py files, showing: line_count filename per line"

# ระบุเงื่อนไข
"Compress all .txt files, but skip any containing 'test'"
```

### คำอธิบายที่ไม่ดี

```bash
# กำกวมเกินไป
"help me process logs"

# ไม่สมเหตุสมผล
"help me write an operating system"
```

## 19.3 รูปแบบคำสั่ง Shell ที่ AI สร้าง

### รูปแบบ 1: คำสั่งเดียว

```bash
# มนุษย์ถาม: หาไฟล์ Python 10 อันดับแรกที่มีบรรทัดมากที่สุด
find . -name "*.py" -exec wc -l {} + | sort -rn | head -10
```

### รูปแบบ 2: Shell Script

```bash
# มนุษย์ถาม: ประมวลผลภาพเป็น batch
cat > process_images.sh << 'EOF'
#!/bin/bash
for img in *.jpg; do
    convert "$img" -resize 800x600 "thumb_$img"
done
EOF
```

## 19.5 ให้ AI ช่วย Debug

```bash
# มนุษย์: ได้ข้อผิดพลาดหลังรันคำสั่งนี้
$ ./deploy.sh
# Output: /bin/bash^M: bad interpreter: No such file or directory

# AI: นี่คือปัญหาของการขึ้นบรรทัดใหม่แบบ Windows รัน:
sed -i 's/\r$//' deploy.sh
```

## 19.6 เครื่องมือทำงานร่วมกันที่แนะนำ

```bash
# tmux: แบ่งหน้าต่าง
```

## 19.7 แบบฝึกหัด

1. อธิบายงานที่ซับซ้อนและให้ AI สร้างคำสั่ง Shell
2. ให้ AI ตรวจสอบสคริปต์ที่มีอยู่เพื่อหาปัญหาด้านความปลอดภัย
3. ใช้การวนรอบเพื่อให้ AI ช่วยเขียนเครื่องมือที่ใช้งานได้จริง
4. ให้ AI ช่วยอธิบายโค้ด Shell ที่ไม่เข้าใจ
