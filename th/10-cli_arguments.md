# 10. การแยกวิเคราะห์อาร์กิวเมนต์และเครื่องมือ CLI

---

## 10.1 พื้นฐานอาร์กิวเมนต์บรรทัดคำสั่ง

```bash
#!/bin/bash

echo "สคริปต์: $0"
echo "อาร์กิวเมนต์แรก: $1"
echo "อาร์กิวเมนต์ที่สอง: $2"
echo "อาร์กิวเมนต์ที่สาม: ${3:-default}"  # ค่าเริ่มต้น
echo "จำนวนอาร์กิวเมนต์: $#"
echo "อาร์กิวเมนต์ทั้งหมด: $@"
```

### การรัน

```bash
./script.sh foo bar
# Output:
# สคริปต์: ./script.sh
# อาร์กิวเมนต์แรก: foo
# อาร์กิวเมนต์ที่สอง: bar
# อาร์กิวเมนต์ที่สาม: default
# จำนวนอาร์กิวเมนต์: 2
# อาร์กิวเมนต์ทั้งหมด: foo bar
```

## 10.2 การแยกวิเคราะห์อาร์กิวเมนต์แบบง่าย

### พารามิเตอร์ตำแหน่ง

```bash
#!/bin/bash

if [[ $# -eq 0 ]]; then
    echo "การใช้งาน: $0 <ไฟล์>"
    exit 1
fi

FILE="$1"

if [[ ! -f "$FILE" ]]; then
    echo "ข้อผิดพลาด: ไฟล์ไม่มีอยู่"
    exit 1
fi

echo "กำลังประมวลผล $FILE..."
```

### จัดการอาร์กิวเมนต์ที่เป็นตัวเลือก

```bash
#!/bin/bash

VERBOSE=false
OUTPUT=""

while [[ $# -gt 0 ]]; do
    case "$1" in
        -v|--verbose)
            VERBOSE=true
            shift
            ;;
        -o|--output)
            OUTPUT="$2"
            shift 2
            ;;
        -*)
            echo "ตัวเลือกที่ไม่รู้จัก: $1"
            exit 1
            ;;
        *)
            FILE="$1"
            shift
            ;;
    esac
done

$VERBOSE && echo "โหมด verbose เปิดอยู่"
[[ -n "$OUTPUT" ]] && echo "ส่งออกไป: $OUTPUT"
```

## 10.3 `getopts`: การแยกวิเคราะห์ตัวเลือกมาตรฐาน

```bash
#!/bin/bash

while getopts "hv:o:" opt; do
    case $opt in
        h)
            echo "ข้อมูลวิธีใช้"
            exit 0
            ;;
        v)
            echo "โหมด verbose: $OPTARG"
            ;;
        o)
            echo "ไฟล์ส่งออก: $OPTARG"
            ;;
        \?)
            echo "ตัวเลือกไม่ถูกต้อง: -$OPTARG" >&2
            exit 1
            ;;
    esac
done

shift $((OPTIND - 1))

echo "อาร์กิวเมนต์ที่เหลือ: $@"
```

### ข้อมูลอ้างอิงรูปแบบตัวเลือก

| รูปแบบ | ความหมาย |
|--------|---------|
| `getopts "hv:"` | `-h` ไม่ต้องมีอาร์กิวเมนต์, `-v` ต้องมีอาร์กิวเมนต์ |
| `OPTARG` | ค่าอาร์กิวเมนต์ของตัวเลือกปัจจุบัน |
| `OPTIND` | ดัชนีอาร์กิวเมนต์ถัดไป |

## 10.4 การรับข้อมูลแบบโต้ตอบ

### `read`: อ่านข้อมูลจากผู้ใช้

```bash
#!/bin/bash

# อ่านพื้นฐาน
read -p "ใส่ชื่อของคุณ: " name
echo "สวัสดี, $name"

# รหัสผ่าน (ซ่อน)
read -sp "ใส่รหัสผ่าน: " password
echo

# อ่านหลายค่า
read -p "ใส่ชื่อและอายุ: " name age
echo "$name อายุ $age ปี"

# หมดเวลา
read -t 5 -p "ใส่ข้อมูลภายใน 5 วินาที: " value
```

### การยืนยัน

```bash
confirm() {
    read -p "$1 (y/n) " -n 1 -r
    echo
    [[ $REPLY =~ ^[Yy]$ ]]
}

if confirm "ลบไฟล์นี้?"; then
    echo "กำลังลบ..."
fi
```

## 10.5 อินเทอร์เฟซเมนู

```bash
#!/bin/bash

PS3="เลือกการทำงาน: "

select choice in "สร้างโปรเจกต์" "ลบโปรเจกต์" "ออก"; do
    case $choice in
        "สร้างโปรเจกต์")
            echo "กำลังสร้าง..."
            ;;
        "ลบโปรเจกต์")
            echo "กำลังลบ..."
            ;;
        "ออก")
            echo "ลาก่อน!"
            exit 0
            ;;
        *)
            echo "ตัวเลือกไม่ถูกต้อง"
            ;;
    esac
done
```

## 10.6 สรุปย่อ

| ไวยากรณ์ | คำอธิบาย |
|---------|-------------|
| `$0` | ชื่อสคริปต์ |
| `$1`, `$2`... | พารามิเตอร์ตำแหน่ง |
| `$#` | จำนวนอาร์กิวเมนต์ |
| `${var:-default}` | ค่าเริ่มต้น |
| `getopts "hv:" opt` | แยกวิเคราะห์ตัวเลือก |
| `$OPTARG` | อาร์กิวเมนต์ของตัวเลือกปัจจุบัน |
| `read -p "ข้อความ:" var` | อ่านข้อมูล |
| `read -s var` | ข้อมูลที่ซ่อน (รหัสผ่าน) |
| `read -t 5 var` | หมดเวลา 5 วินาที |
| `select` | อินเทอร์เฟซเมนู |

---

## 10.7 แบบฝึกหัด

1. เขียนเครื่องมือ CLI ที่รับ `-n` สำหรับจำนวน, `-v` สำหรับ verbose
2. ใช้ `getopts` เพื่อแยกวิเคราะห์ตัวเลือก `-h` (วิธีใช้), `-o` (ไฟล์ส่งออก)
3. เขียนฟังก์ชันยืนยันที่ดำเนินการต่อเมื่อตอบ y
4. สร้างเมนูเครื่องคิดเลขอย่างง่ายด้วย `select`
