# 19. AI + Shell の協調モード

---

## 19.1 新しい形の人と機械の協力

従来のプログラミング：
- 人がキーボードで入力
- 人がマウスでIDEを操作
- 人が実行とテスト

AI時代のプログラミング：
- 人が要件を説明
- AIがShellコマンドとスクリプトを生成
- 人がレビューして実行
- 人とAIが一緒にデバッグ

---

## 19.2 AIに要件を説明する

### 良い説明

```bash
# 明確なタスク
"/var/log 内の100MB以上のすべての .log ファイルを見つける"

# 期待する出力形式を含める
"すべての .py ファイルを行数: ファイル名 行数 の形式で一覧表示する"

# 制約条件を述べる
"すべての .txt ファイルを圧縮するが、'test' を含むものは除外する"
```

### 悪い説明

```bash
# 漠然としすぎ
"ログを処理するのを助けて"

# 非現実的
"OSを書いて"
```

## 19.3 AIが生成するShellコマンドパターン

### パターン1：単一コマンド

```bash
# 人が聞く：行数が最多的トップ10のPythonファイルを見つける
find . -name "*.py" -exec wc -l {} + | sort -rn | head -10
```

### パターン2：Shellスクリプト

```bash
# 人が聞く：画像を一括処理する
cat > process_images.sh << 'EOF'
#!/bin/bash
for img in *.jpg; do
    convert "$img" -resize 800x600 "thumb_$img"
done
EOF
```

## 19.5 AIにデバッグを手伝わせる

```bash
# 人：このコマンドを実行したらエラーが出た
$ ./deploy.sh
# 出力: /bin/bash^M: bad interpreter: No such file or directory

# AI：これはWindowsの改行コードの問題です。以下のコマンドを実行してください：
sed -i 's/\r$//' deploy.sh
```
